Java容器总结

发表于 2019-08-04 | 分类于 后端技术 ， Java基础

一、概述

Java中的容器包括Collection和Map两种，Collection存储着对象的集合，Map存储键值对。
Collection

1.Set

  • TreeSet：底层使用红黑树实现，它可以确保集合元素处于排序状态。但其查找效率不如HashMap。
  • HashSet：底层使用哈希表实现，具有良好的存取和查找性能。大多数情况下我们会使用HashSet。它具有元素无序、不重复、线程不安全的特点。
  • LinkedHashSet：LinkedHashSet为HashSet的子类，其内部利用链表维护元素的插入顺序。

2.List

  • ArrayList：底层使用动态数组实现，有序、可以重复、线程不安全，查找效率高。
  • Vector：与ArrayList类似，但线程安全
  • LinkedList：底层基于双向链表实现，增删效率高，此外，它还常用作栈、队列和双向队列的实现类。

3.Queue

  • LinkedList：实现了Deque双向队列接口，是一个双向队列。
  • Priority：底层使用数组实现，常用来实现堆这种数据结构。

4.集合常见的遍历方式：

  • 调用集合的forEach方法遍历
  • 使用Iterator接口
  • 使用foreach循环遍历

Map

  • HashMap：基于哈希表实现，线程不安全。
  • TreeMap：基于红黑树实现。
  • HashTable：与HashMap类似，但线程安全。现在已不建议使用，而建议使用concurrent包中的线程安全类。
  • LinkedHashMap：是HashMap的子类，内部使用了双向链表维护插入顺序。
